# 마크다운 데이터 분석 및 변환 필요 사항

## 현재 상황

### 마크다운 데이터 형식
```json
{
  "title": "A glass of soju",
  "mp3": "Balad/A-glass-of-soju.mp3",
  "png": "Balad/A-glass-of-soju.png",
  "artist": "Lim Changjung",
  "description": "...",
  "duration": 291
}
```

### 실제 파일명 형식
- **MP3 파일**: `Balad_1.mp3`, `Balad_2.mp3` (하이픈 없음, 언더스코어 + 숫자)
- **이미지 파일**: `A glass of soju.png` (공백 포함, 실제 노래 제목)

## 문제점

1. **MP3 파일명 불일치**
   - 마크다운: `A-glass-of-soju.mp3`
   - 실제: `Balad_1.mp3`
   - **해결 필요**: MP3 파일명을 마크다운 형식으로 변경하거나, 마크다운을 실제 파일명에 맞게 수정

2. **이미지 파일명 불일치**
   - 마크다운: `A-glass-of-soju.png` (하이픈)
   - 실제: `A glass of soju.png` (공백)
   - **해결 필요**: 하이픈을 공백으로 변환하거나, 실제 파일명에 맞게 수정

## 해결 방안

### 옵션 1: 파일명 변경 (권장)
실제 파일명을 마크다운 데이터에 맞게 변경:
- MP3: `Balad_1.mp3` → `A-glass-of-soju.mp3`
- 이미지: `A glass of soju.png` → `A-glass-of-soju.png`

**장점**: 마크다운 데이터를 그대로 사용 가능
**단점**: 모든 파일명 변경 필요

### 옵션 2: 마크다운 데이터 수정
마크다운 데이터의 파일명을 실제 파일명에 맞게 수정:
- MP3: `A-glass-of-soju.mp3` → `Balad_1.mp3`
- 이미지: `A-glass-of-soju.png` → `A glass of soju.png`

**장점**: 파일명 변경 불필요
**단점**: 마크다운 데이터 수정 필요

### 옵션 3: 유연한 매핑 (권장)
실제 파일명과 마크다운 데이터를 매핑하는 JSON 생성:
- `title`을 기준으로 실제 파일명 찾기
- 하이픈/공백/언더스코어를 무시하고 매칭

**장점**: 파일명 변경 불필요, 데이터 수정 불필요
**단점**: 매핑 로직 필요

## 권장 작업 순서

1. **실제 파일명 확인**
   - 모든 장르의 MP3/이미지 파일명 목록 생성
   - 마크다운 데이터의 `title`과 매칭

2. **매핑 테이블 생성**
   - `title` → 실제 MP3 파일명
   - `title` → 실제 이미지 파일명

3. **musicTracks.json 생성**
   - 마크다운 데이터 + 매핑 테이블 사용
   - 실제 파일명으로 `mp3Url`, `imageUrl` 설정
